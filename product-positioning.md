# 旅途中即時協調 PWA — 產品定位文件
> 用途：費米推估、市場規模試算、創業可行性討論

---

## 一句話定位

**競品都在搶「幫你排好行程」，但沒有人在解決「我們已經在日本了，現在人在哪」。**

---

## 產品概述

- **類型**：PWA（Progressive Web App），免下載、免帳號，手機打開連結即用
- **目標情境**：出發後、旅途中的即時協調，不是行前規劃
- **主打功能**：「集合」——任一成員設定集合時間，所有人手機同步顯示倒數計時
- **技術棧**：單一 HTML 檔 + Firebase Realtime Database（即時同步）
- **離線支援**：有 Service Worker，基本功能離線可用

---

## 核心功能清單

| 功能 | 說明 |
|---|---|
| 集合倒數 | 即時廣播集合時間，全員手機同步顯示倒數 |
| 行程總覽 | 每日行程、住宿、餐廳、停車資訊 |
| 群組訊息 | 依日期留言，可編輯刪除 |
| 費用分帳 | 多幣別記帳，自動計算最優還款路徑 |
| 打包清單 | 個人本地勾選，支援自訂項目 |
| 購物清單 | 預設伴手禮清單，標記已購 |

## 差異化定位

### 對比競品

| | 去趣 (chicTrip) | Funliday | **本產品** |
|---|---|---|---|
| 主要用途 | 旅行前規劃、景點發現 | 行程協作規劃 | **旅行中即時協調** |
| 即時集合功能 | 無 | 無 | **有（核心功能）** |
| 需要帳號 | 是 | 是 | **否** |
| 需要下載 App | 是 | 是 | **否（PWA）** |
| 資料量 | 重（社群、內容平台） | 中 | **極輕（數 KB）** |
| 使用時機 | 出發前 | 出發前 | **旅途中** |

### 核心差異

1. **使用時機不同**：解決「已出發」的問題，不搶行前規劃市場
2. **集合功能獨特**：即時廣播倒數，替代 LINE 群組的效率，有「儀式感」
3. **零門檻**：分享連結，成員直接開啟，無需安裝或註冊
4. **輕量可客製**：單一 HTML 檔，每趟旅行一份，用完即棄

## 目標族群

- **主要**：3–10 人揪團出遊（朋友旅行、家族旅行、公司旅遊）
- **地理市場**：台灣出發的國外旅遊（日本為最大宗）
- **技術門檻**：購買者需有基本設定能力（填 Firebase config），成員零門檻

---

## 商業模式（初步方向）

- **銷售方式**：賣模板（HTML 檔 + 設定教學）
- **定價區間**：NT$300–800（一次性，無訂閱）
- **資料處理**：每位使用者自建 Firebase，賣家零接觸使用者資料
- **平台**：Gumroad 或綠界收款
- **維護責任**：無（模板買斷，使用者自負責任）

---

## Firebase 規模估算

Firebase Spark（免費）額度：
- Realtime DB：1 GB 儲存、10 GB/月下載、**100 個同時連線**
- 單趟旅行資料量：< 5 MB（訊息 + 集合 + 費用）
- 單一旅行團人數：平均 5–8 人
- **免費額度可支撐**：約 10–15 個旅行團同時使用，超過需升級（$25/月）

若走「分散式」（每位使用者自建 Firebase）：賣方成本 = 零
